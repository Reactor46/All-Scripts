This script sets "Read" and "Replicate Directory Changes" permissions in the Active Directory (current domain) for the Domain object and the Configuration object.

The script will not modify an existing entry for the profile sync user in the AD. If the user is in the ACL of the Domain object or the Configuration object it will skip this part!

Tested with SharePoint 2013 and a Active Directory based on Windows Server 2012. 

 

Use it like this (supposed the script file is in the current directory):

 

PowerShell
Edit|Remove
powershell 
.".\GrantProfSyncRightsInAD.ps1" -profSyncAccountName "spprofsync@kc-dev.com"
.".\GrantProfSyncRightsInAD.ps1" -profSyncAccountName "spprofsync@kc-dev.com"
